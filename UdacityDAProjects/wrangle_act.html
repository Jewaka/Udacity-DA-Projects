Quality issues ¶

  1. twitter archive table - Remove retweets in the 'text' column i.e. text starting with RT @

  2. twitter archive table - unnecessary in_reply_to_status_id column and in_reply_to_user_id column

  1. twitter archive table - unnecessary retweeted_status_id column, retweeted_status_user_id column and retweeted_status_timestamp column as consideration is on original ratings

  2. twitter archive table - Some dogs have non-dog names, for example: 'a' or 'an', 'such', 'the', 'getting', 'just' etc in the name column

  3. twitter archive table - rating_numerator more than 10

  4. twitter archive table - timestamp is an object rather than a timestamp

  5. image prediction table - p1,p2 and p3 dog breed's names have no standard capital letter or lowercase names

  6. image prediction table - Not all ratings are dog ratings

  7. All tables - Presence of missing data in some tables which leads to differeing number of rows in the different tables. In twitter archive(2356), image prediction(2075) and tweets(2354)

Tidiness issues ¶

  1. twitter archive table - The columns (doggo, floofer, pupper and puppo) are categorical variables yet placed in separate columns hence should be in a single column

  2. tweets table - column named 'id' rather than 'tweet_id' to make it standard as with the other tables

  3. All tables - Merging of the 3 dataframes to have one master table

Insight #2: What is the association between retweet count and favorite count?¶

We calculate Pearson's correlation coefficient to determine the relationship between retweet count and favorite count.

In [46]:
#Calculating the correlation coefficient

def corr_coeff(x,y):
    std_x = (x-x.mean())/x.std(ddof=0)
    std_y = (y-y.mean())/y.std(ddof=0)
    return (std_x*std_y).mean()

corr_coeff(master_df['retweet_count'], master_df['favorite_count'])
